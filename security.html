Security Overview

Security is baked in, not bolted on

FinPay was designed with a security-first architecture from day one. We operate under strict regulatory requirements and voluntarily exceed them, partnering with leading independent auditors to verify our controls continuously.

PCI DSS Level 1 SOC 2 Type II ISO 27001 GDPR Compliant AML / KYC
🔒

Zero Trust Architecture

Every internal request is verified, authenticated, and authorised — regardless of network origin.

🌐

Defense in Depth

Multiple overlapping layers of controls ensure no single point of failure can compromise your data.

✅

Continuous Audits

Third-party penetration testing and internal red team exercises run quarterly, not annually.

Encryption

Military-grade encryption, everywhere

Data is encrypted in transit and at rest using the strongest commercially available standards. Encryption keys are rotated automatically and stored in hardware security modules (HSMs), never in software.

🔐 Data in Transit TLS 1.3 / HSTS
💾 Data at Rest AES-256-GCM
🗝️ Key Management AWS KMS + HSM
💳 Card Data Tokenisation + Vault
✅
No plain-text card data FinPay never stores raw card numbers. All sensitive payment credentials are tokenised at the point of entry and handled exclusively within our PCI DSS Level 1 certified vault.

Infrastructure

Built on resilient global infrastructure

FinPay runs on multi-region, multi-availability-zone cloud infrastructure with automatic failover. We maintain hot standbys across geographically distributed data centres so your payments never go down.

🏢

Multi-Region Deployment

Production workloads span 4 AWS regions across 3 continents for maximum resilience and low latency.

🔄

Automated Failover

Sub-30-second failover to hot standby with zero manual intervention and no data loss.

🔏

Network Isolation

VPCs, private subnets, and strict security groups isolate every tier of the application stack.

🛡️

DDoS Protection

AWS Shield Advanced + Cloudflare provide volumetric and application-layer DDoS mitigation.

ℹ️
SOC 2 Type II Audited Our infrastructure controls are independently verified by a Big 4 auditor annually. Reports available to enterprise customers under NDA.
Monitoring

Always-on security operations

Our Security Operations Centre (SOC) monitors all systems around the clock. Every API call, log stream, and network packet is aggregated into a SIEM for real-time anomaly detection and alerting.

📡

24 / 7 SOC

Dedicated security analysts monitor alerts and respond to incidents in every time zone.

📊

SIEM Platform

All logs are centralised, correlated, and analysed with custom detection rules tuned for fintech threats.

Automated Alerting

Responsible Disclosure

We reward honest researchers

If you discover a security vulnerability in FinPay, please report it responsibly. We commit to acknowledging every valid report within 24 hours and resolving critical issues within 14 days.

📨
Report a vulnerability Email us at security@FinPay.com with a detailed description and steps to reproduce. Please do not publicly disclose findings until we've had a chance to remediate.
