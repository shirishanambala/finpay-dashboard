API Reference

A REST API built for developers

The FinPay API follows REST conventions, uses JSON for all request and response bodies, and authenticates with HTTP Basic Auth using your API keys. SDKs are available for Node.js, Python, PHP, Java, Ruby, Go, and .NET.

  • ◈ Predictable, resource-oriented URLs
  • ◈ Standard HTTP response codes
  • ◈ Idempotency keys on all write operations
  • ◈ Cursor-based pagination on list endpoints
  • ◈ Signed webhooks with HMAC-SHA256

Common developer questions

Is there a sandbox environment for testing?
Yes. Every FinPay account gets a fully isolated test mode with test API keys, simulated payment methods, and test card numbers. No real money moves in test mode. Switch between live and test mode from your dashboard at any time.
Which SDK languages are supported?
FinPay maintains official SDKs for Node.js, Python, PHP, Java, Ruby, Go, and .NET. iOS and Android SDKs are available for native mobile integrations. All SDKs are open source on GitHub and updated with every API version release.
How do I handle payment verification to prevent fraud?
After a payment is completed on the client, always verify the payment signature on your server using your API secret key before fulfilling the order. The FinPay signature verification docs and SDK helpers walk you through this process for every supported language.
Does FinPay support API versioning?
Yes. FinPay follows a date-based versioning scheme. Your integration is pinned to the API version active when your account was created. Breaking changes are only introduced in new versions, and you'll receive advance notice before any version is deprecated.
Where can I see API uptime and incident history?
FinPay publishes a public status page at status.finpay.in with real-time uptime metrics and a full incident history. You can subscribe to incident alerts via email or webhook.
